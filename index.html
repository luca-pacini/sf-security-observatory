What it does

  • Identity & access

    Reviews user identity configuration and access settings to surface risky or misaligned posture.

  • OAuth & Connected Apps

    Inspects OAuth grants and Connected App configuration for over-broad or stale authorisations. Existing Connected Apps remain in scope — new integrations should use External Client Apps from Spring '26.

  • API configuration

    Checks API access settings and integration surface area against least-privilege expectations.

  • Named Credentials & external endpoints

    Examines Named Credentials and outbound endpoints so external callout configuration stays accountable.

  • Permissions

    Maps permission sets and profile grants to highlight excessive or unexpected privilege.

  • Licences

    Reports on licence allocation to keep entitlement and access aligned across the org.

20+ automated checks. Results stored natively in your org. Nothing phoned home.

Benchmark alignment

Observatory is being aligned to the Security Benchmark for Salesforce (SBS) — an open, vendor-neutral set of controls for Salesforce security. SBS alignment is a P0 v1 priority, delivered as local metadata, static mappings and a coverage view once validated.

  • Explainable evidence. Cards show source, coverage status, posture, verification path, limitation and Observatory value-add where implemented.
  • V1 SBS foundation. Static scanner-to-SBS mapping, local SBS metadata and coverage visibility are v1 P0 work, not post-v1 scope.
  • Coverage view. Target v1 coverage states are Automated, Partial Evidence, Manual Required, Not Covered and Extended Check.
  • No compliance claim. Observatory will not present SBS alignment as certification or formal compliance.

Evidence-first roadmap — not a certification or formal compliance claim. See the roadmap for what's automated today versus planned.

How it works

  1. Request access below.
  2. Deploy the source to your sandbox via SFDX using the install guide.
  3. Configure the self-callout Named Credential (one-time, per the setup guide).
  4. Run a scan and view your posture dashboard.

v1 ships as an SFDX source deploy with a guided install. An unlocked package will follow once distribution stabilises.

How it stays safe

  • Read-only by design. No remediation, no automatic changes to your org. Findings are advisory.
  • Runs in your org. All scanning executes as Apex queueables in your tenant. There is no external service.
  • No outbound destinations. The only callouts are to your own Tooling API via a self-callout Named Credential that you configure and authenticate.
  • No secrets stored. Tokens, session IDs, certificate bodies and raw header values are never persisted or displayed. Sanitised evidence only.
  • Least-privilege scopes. The setup requests only the OAuth scopes needed for Tooling API reads. No write, no impersonation.
  • Kill switch documented. Revoke the principal at any time; scanning degrades gracefully with no fake results.
