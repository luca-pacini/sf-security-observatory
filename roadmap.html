Roadmap

Read-only Salesforce security evidence, mapped to the Security Benchmark for Salesforce (SBS), with compound-risk correlation and native blind-spot visibility.

v1 status: in active development. Last reviewed 2026-06-16. Independent reference implementation — not affiliated with Salesforce or the Security Benchmark for Salesforce project.

At a glance

  • v1 — now

    SBS-aligned evidence, coverage map, retention discipline.

  • v1.1 — next

    SBS lifecycle, version reconciliation, native capability expansion.

  • v1.2 — later

    Additional scanners, deeper posture coverage.

  • v2 — future

    Governance, attestation and accepted risk, entitlement comparison.

What Observatory adds

A feature stays in scope only if it does at least one of the following. Everything else is removed, consolidated, or left as a link to Salesforce Setup.

  • Benchmark coverage. Maps observed evidence to SBS control IDs.
  • Compound-risk correlation. Combines signals across users, permissions, OAuth, API access, sessions, endpoints, certificates, and licences.
  • Native blind-spot visibility. Shows when a Salesforce-native capability is unavailable, disabled, or not configured.
  • Drift and comparison. Shows what changed between retained scans.

Where Salesforce already provides the core native capability — Health Check, Security Center, Event Monitoring, Transaction Security — Observatory links to it or summarises it instead of duplicating it.

v1 — now

v1 focuses on trustworthy, explainable, short-term operational evidence.

  • Dashboard clarity. Every card shows its source, SBS mapping or Extended Check, coverage, and posture.
  • SBS-aligned evidence. Scanner findings are mapped to SBS controls where applicable.
  • Coverage map. Shows Automated, Partial Evidence, Manual Required, Not Covered, and Extended Check.
  • Failure visibility. Scanner failures show as Unknown/Error, never as a false Pass.
  • Retention discipline. Salesforce retains only the latest 3 completed scans.
  • Safe export. CSV export is available for retained evidence and protected against formula injection.
  • Scan comparison. If compared scans used different installed SBS versions, Observatory shows a version-mismatch warning.
  • Local SBS metadata. Ships with bundled SBS metadata and supports controlled admin upload of updated SBS XML.
  • Native blind spots. Cards show when a native Salesforce capability is unavailable, disabled, not configured, or outside the current scan scope.
  • No remediation. Observatory reports posture only. It does not change security settings.

What v1 is not

  • No automatic remediation. No token revocation, API blocking, permission removal, endpoint change, certificate change, or key rotation.
  • No AI summaries. No AI or Gemini advisory summaries.
  • No Event Monitoring ingestion. No high-volume event-log ingestion.
  • Not a Security Center clone. No attempt to replace Salesforce-native security products.
  • No external warehouse sync. Long-term evidence storage is customer-owned and outside the app.
  • No scheduled scans. v1 is manual.
  • No formal compliance claim. SBS-aligned evidence and coverage visibility, not certification.

v1.1 — next

SBS lifecycle management, safer version comparison, and native capability expansion.

  • SBS version reconciliation. Show controls added, removed, renamed, or changed between installed SBS versions.
  • Safer cross-version comparison. Compare scans run under different SBS versions without a silent control-by-control mismatch.
  • SBS upload history. Expand the upload log into a richer version history.
  • SBS registry rollback. Support rollback using retained registry snapshots.
  • Native capability register. Expand Setup paths, licence dependencies, posture, and limitations per card.
  • Connected App and External Client App posture. Improve visibility where metadata access is safe and reliable.
  • API Access Control posture. Detect and explain posture where feasible, without remediation.
  • Storage estimate improvement. Improve retained evidence footprint estimates.
  • Source-org awareness. Harden scan and comparison labels for future multi-org support.

v1.2 — later

Additional scanner coverage, after v1 and v1.1 stability work is complete.

  • Public Content links. Optional scanner for public file or link exposure.
  • Field History Tracking. Optional scanner for configured sensitive-field lists.
  • Sites and Experience depth. Expand baseline exposure checks where safe.
  • Transaction Security. Inventory only, no blocking or remediation.
  • Permission source depth. Improve Profile, Permission Set, and Permission Set Group risk visibility.
  • Licence alignment. Improve over-licensed, missing-access, and unexpected-access views.
  • Historical baselines. Daily, weekly, and monthly labels, if storage-safe.

v2 — future

Governance evidence and formal review workflows.

  • Manual attestation. Governance evidence for controls that cannot be scanned.
  • N/A justification. Explain controls that do not apply.
  • Accepted risk. Owner, expiry, justification, and review status.
  • Functional role entitlement comparison. Compare expected access against actual access by role.
  • Historical drift reporting. Repeated, worsened, improved, and resolved posture over time.
  • External reporting model. Replication-ready reporting objects or views.
  • Framework export. Export SBS-aligned evidence for wider governance reporting.

Public safety

Everything shown in this project's public materials — examples, screenshots, and sample exports — comes from a development org using synthetic data, never customer data, org secrets, credentials, tokens, or real scan evidence.
